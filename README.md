## Описание работы: 
``` text
Модуль 2: Контроллеры SDN и программирование сетей

Тема: Архитектура SDN-контроллеров (ONOS, ODL). Программирование с помощью Northbound API (REST).

Задание 2: "Умная маршрутизация для офиса AgroTex" (Командное)

Цель: Научиться программировать сеть через API контроллера для реализации специфической бизнес-логики.

Задача для команд:

Команда 1 (Сценарий "Безопасность"): Реализовать политику "изоляция гостевой сети". В топологии Mininet создать две подсети: Рабочая (Hosts 1-2) и Гостевая (Hosts 3-4). Написать приложение для контроллера ONOS (используя REST API), которое запрещает любой трафик из Гостевой сети в Рабочую, но разрешает доступ в Интернет (условно, через хост-шлюз).

Команда 2 (Сценарий "Качество обслуживания - QoS"): Реализовать приоритезацию голосового трафика. Создать топологию, где есть источник VoIP-трафика (пометить его DSCP значением, например, EF) и обычный трафик. Написать приложение, которое с помощью потоковых правил направляет VoIP-трафик по пути с меньшей задержкой (например, через более скоростной линк, если смоделировать его).

Результат:

Презентация на 5–7 минут, где команда показывает работу своего приложения.

Исходный код приложения (на Python) в репозитории GitHub/GitLab.

Демонстрация работы через ping, iperf и т.д.
```
